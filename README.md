# ? UsuhiWeb - Веб-приложение для заказа японской кухни

Современное веб-приложение для онлайн-заказа суши, роллов, напитков и других блюд японской кухни, разработанное на Django.

## ? Описание
UsuhiWeb - это полнофункциональная платформа электронной коммерции, специализирующаяся на японской кухне. Приложение предоставляет интуитивно понятный интерфейс для просмотра меню, управления корзиной покупок и оформления заказов.

## ? Основные возможности

### Для пользователей:
- ? **Каталог блюд** - просмотр меню с категориями (суши, роллы, сеты, напитки и др.)
- ? **Корзина покупок** - добавление/удаление товаров с динамическим обновлением
- ? **Адаптивный дизайн** - корректное отображение на всех устройствах
- ? **Система авторизации** - регистрация, вход, восстановление пароля
- ??? **Личный кабинет** - управление профилем и историей заказов
- ? **Оформление заказов** - выбор способа оплаты и доставки
- ? **Фильтрация по категориям** - быстрый поиск нужных блюд

### Для администраторов:
- ? **Админ-панель Django** - управление товарами, заказами, пользователями
- ? **CRUD операции** - добавление, редактирование, удаление блюд
- ? **Мониторинг заказов** - отслеживание статуса заказов

## ? Технологии

## ? Установка и запуск

### Предварительные требования:
- Python 3.8+
- pip (менеджер пакетов Python)
- Git

### Пошаговая установка:

1. **Клонирование репозитория:**
   ```bash
   git clone https://github.com/Beksh0800/UsuhiWeb.git
   cd UsuhiWeb
   ```

2. **Создание виртуального окружения (рекомендуется):**
   ```bash
   python -m venv venv
   
   # Windows
   venv\Scripts\activate
   
   # macOS/Linux
   source venv/bin/activate
   ```

3. **Установка зависимостей:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Применение миграций базы данных:**
   ```bash
   python manage.py makemigrations
   python manage.py migrate
   ```

5. **Создание суперпользователя (опционально):**
   ```bash
   python manage.py createsuperuser
   ```

6. **Сбор статических файлов:**
   ```bash
   python manage.py collectstatic
   ```

7. **Запуск сервера разработки:**
   ```bash
   python manage.py runserver
   ```

Приложение будет доступно по адресу: `http://127.0.0.1:8000/`

## ? Модели данных

- **Food** - информация о блюдах (название, категория, цена, описание)
- **Cart** - корзина пользователя
- **CartItem** - элементы корзины
- **Order** - заказы
- **Profile** - профили пользователей

## ? Основные URL маршруты

- `/` - главная страница с меню
- `/about/` - страница "О нас"
- `/cart/` - корзина покупок
- `/profile/` - личный кабинет
- `/login/` - авторизация
- `/signup/` - регистрация
- `/admin/` - админ панель

## ? Конфигурация

### Переменные окружения:
Создайте файл `.env` в корне проекта:
```
DEBUG=True
SECRET_KEY=your-secret-key-here
DATABASE_URL=sqlite:///db.sqlite3
```

### Настройки Django:
Основные настройки находятся в `UsuhiWeb/settings.py`

## ? Развертывание в продакшене

### Подготовка к продакшену:
1. Установите `DEBUG = False` в settings.py
2. Настройте `ALLOWED_HOSTS`
3. Используйте PostgreSQL вместо SQLite
4. Настройте статические файлы через nginx
5. Используйте gunicorn как WSGI сервер

### Пример развертывания с gunicorn:
```bash
pip install gunicorn
gunicorn UsuhiWeb.wsgi:application --bind 0.0.0.0:8000
```
